<h2>Composer - Enforcing GIT hooks</h2> <p><a href="https://getcomposer.org/">Composer</a> allows you to use custom plugins to add extra behaviour to install, update, uninstall, ... events. The first time I &quot;discovered&quot; this extra functionality was during a <a href="https://speakerdeck.com/veewee/assuring-code-quality-with-grumphp-phpleuven12">talk</a> by Toon Verwerft (<a href="https://twitter.com/toonverwerft">@toonverwerft</a>) about <a href="https://github.com/phpro/grumphp">GrumPHP</a>, a PHP QA tool.</p> <h3>GrumPHP</h3> <p>GrumPHP is a Composer plugin which installs a couple of GIT hooks in your repository, which perform QA checking on your project.</p> <p>The copying of the GIT hooks is actually pretty easy using a plugin, you just attach event handlers to certain Composer package/script events such as install, update and uninstall.</p> <h3>Implementation example</h3> <p>I made two repositories which display how easily you can enforce git hooks for all your projects by requiring 1 extra package.</p> <p>Check <a href="https://github.com/TSchuermans/enforce-hooks">TSchuermans/enforce-hooks</a> for the source code of the plugin (very basic stuff) and <a href="https://github.com/TSchuermans/enforce-hooks-implementation">TSchuermans/enforce-hooks-implementation</a> for an implementation example of said plugin.</p>
